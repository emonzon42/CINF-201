How to Link JS to your Page

This page demonstrates how to link JavaScript to your HTML. To view the code, right-click the page and select "View Page Source." The JS will be located in a few places. One place is in the opening tag of the first button as an attribute. Another place will be inside of the first set of <script></script> tags. It will also be inside of an external JS file which can be found by clicking the src attribute value linked in the script tag just before the closing body tag. JavaScript can be linked in one of three ways:

  1. Inline
  2. Internally
  3. Externally

Inline Linking

Inline JavaScript is done by placing the event handler (trigger) inside the HTML tag as an attribute. The button below has an onclick attribute which I use to trigger an alert function. The button is the object, onclick is the event, and alert is the function which we trigger. Click it to test it out.

Internal Linking

Linking JavaScript internally requires the usage of <script></script> tags in your HTML document. Since the JavaScript isn't directly attached to the HTML tag through an attribute, we must link to it some other way. One popular method is to use document.getElementById().

The document object has a few functions or methods that can be used to search through the HTML for an element. getElementById is one such method and it requires the id of an element on the page. Whatever id we provide for the getElementById function must match the id of some element on the page, or the function will not work.

External Linking (Best Practice)

Linking JavaScript externally also requires the usage of <script></script> tags in your HTML document. However, you don't place any JS between the tags. Instead, you add a src attribute to the opening tag and then provide the path to your JS file.

If it's in the same folder, you simply name the file with a .js extension. However, if it's in some other folder, you need to name that folder and follow it by a forward slash ("/") and then the file name.
