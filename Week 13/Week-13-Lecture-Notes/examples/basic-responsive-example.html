Responsive/Unresponsive Examples

Please go through the examples below for demonstrations of how pixels/percentages can affect the responsiveness of a page.

Unresponsive Example 1

The divs inside of the div below are unresponsive and don't work well on mobile devices. The reason for this is because I have used fixed pixel amounts for widths. When the screen size is smaller than the pixel width of a container, a horizontal bar is created to let users scroll and see the content. This is not ideal for users because it isn't an expected behavior for web pages.

Responsive Example 1 (Floats)

The elements below are responsive and initially set to 50% of their parent container. When the page is above 900px, the divs will have a blue background. Between 600px and 899px, they will have a green background. Any screen 599px or lower will display red divs but in a one-column layout. Adjust the size of your browser window horizontally to see how it changes. You can see the CSS by inspecting the page or viewing the source after right-clicking it.
